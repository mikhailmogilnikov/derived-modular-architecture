## 按框架

每份指南使用相同的 mini-shop，但组合根（composition root）不同：

| 指南 | 组合根 | 仓库中的示例 |
| --- | --- | --- |
| [Vite + React](/guides/vite-react) | `src/app/` | 可运行的 canon |
| [Next.js](/guides/nextjs) | `src/app/` | App Router + RSC |
| [Astro](/guides/astro) | `src/pages/` + `src/app/` | `.astro` frontmatter |
| [SvelteKit](/guides/sveltekit) | `src/routes/` | Svelte 5 runes |
| [Vue + Vite](/guides/vue) | `src/app/` | SFC + composables |

所有示例均通过 `dma check`——源码位于 [examples](https://github.com/mikhailmogilnikov/derived-modular-architecture/tree/main/examples)。

## 各栈共通点

- 路由保持精简——只从 `*/public/*` 导入
- 禁止 `feature → feature`
- `services/` 在提升（promotion）时创建，而非提前搭建
- CI：`dma check` + 自选 linter
