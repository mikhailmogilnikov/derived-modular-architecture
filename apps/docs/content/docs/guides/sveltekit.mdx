**SvelteKit + Svelte 5** example: breadth across features, promoted cart service, event wiring in `app-providers.svelte`.

Sources: [examples/sveltekit-routes](https://github.com/mikhailmogilnikov/derived-modular-architecture/tree/main/examples/sveltekit-routes).

## Composition root = routes/

DMA treats `src/routes/` as `app/`:

```text
src/routes/
‚îú‚îÄ‚îÄ +layout.svelte       # import app.css, mount AppProviders
‚îú‚îÄ‚îÄ +page.svelte         # thin shell ‚Äî wiring only
‚îî‚îÄ‚îÄ app-providers.svelte # theme tokens + event subscriptions
```

`+page.svelte` mounts `features/*/public/*` and stage-0 `profile.svelte`, `promo.svelte`.

## Event wiring

Checkout emits an event; app subscribes and calls a service:

```svelte
<!-- routes/app-providers.svelte -->
<script>
  import { onCheckoutCompleted } from "@/features/checkout/public/checkout.events";
  import { clearCart } from "@/services/cart/public/cart";

  onCheckoutCompleted(() => clearCart());
</script>
```

No `checkout ‚Üí cart` as a feature edge ‚Äî app imports both modules downward. See [Cross-module wiring](/guides/cross-module-wiring).

## Mini-shop flow

1. **Catalog** ‚Äî `catalog.store.ts` + `addCartItem` from cart service
2. **Checkout** ‚Äî cart service + `checkout.store.ts` (payment status)
3. **AppProviders** ‚Äî `checkoutCompleted` ‚Üí `clearCart()`
4. **Profile / Promo** ‚Äî stage-0, shared `Card`

## Svelte 5 runes

| File | Runes |
| --- | --- |
| `profile.svelte` | `$state` |
| `app-providers.svelte` | `$props`, `$effect` |
| `button.svelte` | `$props` |

Stores in `*.store.ts` ‚Äî regular Svelte stores, auto-subscription via `$store`.

## Where to put code

| Question | Place |
| --- | --- |
| Route shell, layout? | `src/routes/` |
| UI flow? | `features/<name>/public/*` |
| Single-feature state? | `features/<name>/<name>.store.ts` |
| 2+ modules? | `services/<name>/public/*` |
| Portable helper/UI? | `shared/lib/`, `shared/ui/` |
| Subscribers / ports? | `routes/app-providers.svelte` |

Internals are imported **relative** from the `public/` file of the same feature.

## Styles

| Kind | Where |
| --- | --- |
| Global reset | `src/app.css` ‚Üí `+layout.svelte` |
| Design tokens | `app-providers.svelte` (`--color-*` on `.app-theme`) |
| Route shell | scoped `<style>` in `+page.svelte` |
| Feature page | scoped in `public/*.svelte` |
| Shared UI | scoped in `shared/ui/*.svelte` |
| Feature sheet | `checkout/checkout.css` ‚Üí import from `public/` |

Tokens in the provider shell ‚Äî features and `shared/ui` read `var(--‚Ä¶)` without importing each other.

## Tests

Colocated `*.test.ts` (Vitest):

- `catalog.store.test.ts` ‚Äî store + `svelte/store`
- `services/cart/public/cart.test.ts` ‚Äî service API
- `shared/lib/format-currency.test.ts` ‚Äî pure helper

`dma check` scans `*.test.ts` too ‚Äî imports in tests follow the same rules. In the example, tests are colocated and import relative inside the module or via `public/`.
