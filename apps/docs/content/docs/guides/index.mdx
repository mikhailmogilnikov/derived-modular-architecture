## By framework

Each guide uses the same mini-shop with a different composition root:

| Guide | Composition root | Example in repo |
| --- | --- | --- |
| [Vite + React](/guides/vite-react) | `src/app/` | Runnable canon |
| [Next.js](/guides/nextjs) | `src/app/` | App Router + RSC |
| [Astro](/guides/astro) | `src/pages/` + `src/app/` | `.astro` frontmatter |
| [SvelteKit](/guides/sveltekit) | `src/routes/` | Svelte 5 runes |
| [Vue + Vite](/guides/vue) | `src/app/` | SFC + composables |

All examples pass `dma check` — sources live in [examples](https://github.com/mikhailmogilnikov/derived-modular-architecture/tree/main/examples).

## Common across stacks

- Routes stay thin — import only from `*/public/*`
- No `feature → feature`
- `services/` is created on promotion, not upfront
- CI: `dma check` + linter of your choice
